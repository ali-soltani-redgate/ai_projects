# 3.1.1 Prompt Engineering

> Each task follows the same pattern: **understand the concept → implement it in code**.  
> One or two tasks per day. Each should take 30–60 minutes.

---

## 1. Zero-Shot Prompting

- [ ] **Understand:** zero-shot means no examples — instruction only. The model relies entirely on pre-trained knowledge.
- [ ] **Implement:** write a Python script that sends zero-shot prompts for 5 different task types (summarise, classify, translate, extract, rewrite) and prints the results.
- [ ] **Implement:** run the same task with 3 different instruction phrasings and log which wording produces the best output.
- [ ] **Reflect:** add a comment in your script noting when zero-shot was sufficient and when it fell short.

---

## 2. Few-Shot Prompting

- [ ] **Understand:** few-shot works by showing the model input→output patterns directly in the prompt (in-context learning).
- [ ] **Implement:** write a `build_few_shot_prompt(task, examples: list[dict]) -> str` function that dynamically assembles a prompt from a list of `{"input": ..., "output": ...}` dicts.
- [ ] **Implement:** run a sentiment classification task with 1, 3, 5, and 10 shots. Log accuracy manually on 10 test cases for each.
- [ ] **Implement:** deliberately use low-quality or contradictory examples and observe how the output degrades. Log your findings.

---

## 3. System Prompts

- [ ] **Understand:** system prompts set persistent context, persona, and constraints before the user message. Handled differently across APIs.
- [ ] **Implement:** write a `make_client(system_prompt: str)` wrapper (or a simple function) that injects a system prompt into every call, so callers never repeat it.
- [ ] **Implement:** write a system prompt that assigns a persona and restricts output to bullet points. Test it with 5 different user messages.
- [ ] **Implement:** write the same task twice — once with a system prompt, once without — and print both responses side by side to compare.

---

## 4. Chain-of-Thought (CoT)

- [ ] **Understand:** CoT forces the model to externalise intermediate reasoning steps, which reduces errors on multi-step problems.
- [ ] **Implement:** write a `cot_prompt(question: str) -> str` function that appends "Let's think step by step." and call it on 3 logic/maths problems.
- [ ] **Implement:** write a manual CoT few-shot prompt where each example includes explicit reasoning steps. Compare output quality against the automatic trigger.
- [ ] **Implement:** build a comparison script: same 5 questions sent with CoT and without. Print answers side by side. Note where CoT helped and where it added noise.

---

## 5. Output Format Control

- [ ] **Understand:** models can reliably follow output schemas when the format is shown explicitly in the prompt.
- [ ] **Implement:** write a prompt that requests JSON output with a fixed schema `{"label": str, "confidence": float, "reason": str}`. Parse the response with `json.loads()` and handle parse errors.
- [ ] **Implement:** define the same schema as a Pydantic model and validate the parsed JSON against it. Print validation errors when the model deviates.
- [ ] **Implement:** use XML tags (`<answer>`, `<reasoning>`) to separate parts of a complex response. Write a parser that extracts each tag's content using regex or an XML library.

---

## 6. Role & Persona Assignment

- [ ] **Understand:** role assignment shifts the model's tone, vocabulary depth, and framing — it does not change factual knowledge.
- [ ] **Implement:** write a `role_prompt(role: str, audience: str, task: str) -> str` factory that composes a prompt from those three parts.
- [ ] **Implement:** send the same technical explanation request with three roles: "senior engineer", "junior developer", "non-technical manager". Print responses and compare depth and language.

---

## 7. Advanced Patterns

- [ ] **Understand:** self-consistency samples the model multiple times and takes the majority answer — reduces variance on reasoning tasks.
- [ ] **Implement:** write a `self_consistent_answer(question: str, n: int = 5) -> str` function that calls the model `n` times and returns the most common answer.
- [ ] **Understand:** ReAct (Reason + Act) interleaves thinking and tool calls — this is the foundation of agent loops.
- [ ] **Implement:** simulate a ReAct loop manually: write a prompt that asks the model to output `Thought:` / `Action:` / `Observation:` turns, parse those turns, and feed the next one back in a loop. Use a mock "tool" (e.g., a dict lookup) for the action.

---

## 8. Common Pitfalls

- [ ] **Understand + Implement:** take a vague prompt that produces inconsistent output. Rewrite it to be precise and run both 5 times. Log how consistency changes.
- [ ] **Understand + Implement:** write a prompt using "do NOT mention X". Observe failures. Rewrite it positively ("only discuss Y") and compare.
- [ ] **Understand + Implement:** write a system prompt with 10 rules. Deliberately violate some in a user message and observe which rules get dropped. Reduce to the 3 most important rules and retest.
- [ ] **Understand + Implement:** simulate a prompt injection attack — put an instruction inside a "user document" that tries to override your system prompt. Implement a basic sanitisation step.

---

## 9. Prompt Iteration Workflow

- [ ] **Understand:** prompts are code — they need versioning, testing, and change logs.
- [ ] **Implement:** create a `prompts.py` file that stores prompt templates as versioned constants (e.g. `SUMMARISE_V1`, `SUMMARISE_V2`). Never hardcode prompts inline.
- [ ] **Implement:** write a `run_eval(prompt_fn, test_cases: list[dict]) -> dict` function that runs a prompt against labelled test cases and returns an accuracy score. Run it against two versions of the same prompt.

---

## 10. Mini Project — Prompt Strategy Comparison

- [ ] **Implement:** pick one task (e.g. extract key facts from a paragraph). Implement all 4 strategies in one script:
  1. Zero-shot
  2. Few-shot (3 examples)
  3. Chain-of-thought
  4. Structured JSON output
- [ ] **Implement:** run all 4 against the same 5 inputs. Print results in a table.
- [ ] **Reflect:** write 5–10 lines in a comment or separate `.md` note on what surprised you and which strategy you'd default to for this task type.
